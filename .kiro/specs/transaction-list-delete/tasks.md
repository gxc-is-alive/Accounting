# 实现计划：交易记录列表删除功能优化

## 概述

本实现计划将在移动端交易记录列表中添加直接删除按钮，使用户无需进入详情面板即可快速删除交易记录。

## 任务

- [x] 1. 修改 TransactionCard 组件添加删除按钮
  - 在卡片右侧添加删除图标按钮
  - 使用 `el-icon` 和 `Delete` 图标
  - 添加 `@delete` 事件发射
  - 使用 `@click.stop` 阻止事件冒泡
  - 确保按钮触摸区域至少 44x44px
  - 使用危险色样式
  - _需求：1.1, 2.1, 2.2, 2.3, 2.4, 6.3_

- [x] 2. 更新 TransactionList 页面使用新的删除功能
  - 确保 TransactionCard 的 `@delete` 事件正确绑定到 `handleDelete` 函数
  - 验证删除确认对话框正常显示
  - 验证删除成功后列表刷新
  - 验证详情面板的删除功能仍然正常工作
  - _需求：1.2, 1.3, 1.4, 1.5, 4.1, 4.2, 4.3_

- [x] 3. 添加响应式样式和布局优化
  - 确保删除按钮在不同屏幕尺寸下正常显示
  - 确保按钮不与其他元素重叠
  - 添加 hover 和 active 状态样式
  - 优化触摸设备上的点击体验
  - _需求：6.1, 6.2, 6.4_

- [x] 4. 测试和验证
  - 测试移动端删除功能
  - 测试桌面端功能未受影响
  - 测试详情面板删除功能
  - 测试错误处理场景
  - 测试不同屏幕尺寸
  - _需求：3.1, 3.2, 3.3, 5.1, 5.2, 5.3, 5.4, 5.5_

## 注意事项

- 保持桌面端功能完全不变
- 确保事件冒泡被正确阻止
- 删除按钮应该清晰可见但不突兀
- 遵循 Vue 3 Composition API 最佳实践
- 使用现有的设计系统颜色和间距变量
